# ❌⭕ Tic-Tac-Toe Online Engine

**Tic-Tac-Toe Online Engine** — это современная игровая платформа для игры в «Крестики-Нолики», объединяющая классический геймплей с продвинутыми технологиями: от непобедимого ИИ на базе теории игр до мультиплеера в реальном времени.

---

## ✨ Ключевые возможности

### 🎮 Игровые режимы
* **PvE (Player vs Environment):** Сражение с умным ботом. Используется алгоритм **Minimax**, который просчитывает все возможные исходы, делая бота практически непобедимым.
* **PvP (Player vs Player):** Онлайн-матчи с реальными людьми. Система поддерживает создание приватных комнат и общее лобби через WebSocket.

### 🛠 Технологический фундамент
* **Real-time Gameplay:** Мгновенная синхронизация ходов через **WebSockets (STOMP)**. Никаких перезагрузок страницы во время игры.
* **Session Management:** Возможность прервать игру с ботом и продолжить её позже благодаря сохранению состояний в БД.
* **Matchmaking Lobby:** Интерактивный список активных онлайн-игр с фильтрацией по статусу (Ожидание/В игре).

### 👤 Личный кабинет и статистика
* **Gamified Profiles:** Индивидуальная статистика (Wins/Losses/Draws) и автоматический расчет Win Rate.
* **Identity:** Система загрузки и удаления аватаров (Multipart upload), а также кастомизация игрового имени.
* **Leaderboard:** Глобальный рейтинг лучших игроков.
* **Game History:** Детальный архив всех сыгранных партий.

### 🔐 Безопасность
* **JWT Authentication:** Защищенный вход и регистрация. Токены хранятся в **HttpOnly Cookies**, что предотвращает XSS-атаки.
* **Spring Security:** Тонкая настройка прав доступа и защита API-эндпоинтов.

## 🏗 Архитектура системы

Проект следует принципам **Layered Architecture** (Многослойная архитектура):



1.  **Presentation Layer (Frontend):**
    * **FreeMarker (SSR):** Динамическая генерация страниц на стороне сервера.
    * **Tailwind CSS:** Современная адаптивная верстка.
    * **Vanilla JS:** Легковесная логика взаимодействия с WebSocket и API.
2.  **Service Layer (Business Logic):**
    * Ядро игры, реализующее правила и алгоритм ИИ.
    * WebSocket-контроллеры для обработки потока игровых событий.
3.  **Data Layer (Persistence):**
    * **Spring Data JPA:** Работа с базой данных PostgreSQL/H2.
    * Кастомные мапперы и конвертеры для хранения игровых полей.



---

## 🤖 Алгоритм ИИ: Minimax

Бот оценивает каждое состояние игрового поля, выстраивая дерево решений. Математическая модель оценки:
* **Победа бота:** +10
* **Ничья:** 0
* **Победа игрока:** -10

Бот всегда выбирает путь, минимизирующий максимально возможный выигрыш противника (отсюда и название — Minimax).

## 🚀 Быстрый запуск (Gradle)

### Требования
* **Java 17** (или выше)
* **Gradle 7.x+** (или встроенный Gradle Wrapper)
* **PostgreSQL** (или использование встроенной H2)

### Инструкция по запуску

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/ilgiz41/Web-tictactoe.git
    cd Web-tictactoe
    ```

2.  **Настройте окружение:**
    Отредактируйте `src/main/resources/application.properties`, указав параметры вашей БД и секретные ключи для JWT.

3.  **Соберите проект:**
    ```bash
    ./gradlew build
    ```

4.  **Запустите приложение:**
    ```bash
    ./gradlew bootRun
    ```
    
6.  **Играйте:** Откройте в браузере [http://localhost:8080](http://localhost:8080).
